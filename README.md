# Automated Fairness Auditing Pipeline

Companion code for the paper:

> **Automated Fairness Auditing and Bias Detection in Machine Learning via Synthetic Data and Explainable AI**
> Monis Ahmed Khan, Chase Martin, Razib Iqbal, Rahul Dubey
> Department of Computer Science, Missouri State University

This repository is the **implementation reference** for the paper. The
six-stage pipeline (preprocessing, synthetic data, model training,
fairness evaluation, XAI diagnostics, audit reporting) is implemented
in full, every supporting library is wired up, and every figure and
table from the manuscript is produced by code in this repo.

## The 6-Stage Pipeline

| Stage | Module                          | What it does |
|-------|---------------------------------|--------------|
| 1 | `pipeline/stage1_preprocess.py` | Median/mode imputation, label encoding, z-score scaling, stratified train/test split |
| 2 | `pipeline/stage2_synthetic.py`  | SMOTE (`imbalanced-learn`), counterfactual flip on protected attributes, GAN-based synthesis (CTGAN via `sdv` if installed; Gaussian-copula fallback otherwise) |
| 3 | `pipeline/stage3_models.py`     | `sklearn` Logistic Regression / Random Forest, real-data-only test evaluation |
| 4 | `pipeline/stage4_fairness.py`   | DPD, DIR, EOD with two-proportion z-test for significance, conservative any-fail decision rule |
| 5 | `pipeline/stage5_diagnostics.py`| SHAP feature attributions + per-group disparity flags, LIME on counterfactual pairs |
| 6 | `pipeline/stage6_reporting.py`  | Plain-language + pass/fail audit report (JSON + Markdown) |

## The 7 Benchmark Datasets

All loaders are in `pipeline/datasets.py`. Real datasets are downloaded
into `data/` on first use; gated datasets (MEPS, CivilComments) fall
back to faithful synthetic stand-ins so the pipeline runs end-to-end
out of the box.

| Dataset              | Source                               | Notes |
|----------------------|--------------------------------------|-------|
| UCI Adult            | UCI ML repo                          | sex, race |
| COMPAS               | propublica/compas-analysis           | race, sex |
| Diabetes 130-US      | UCI ML repo                          | race, gender |
| MEPS                 | synthetic stand-in                   | AHRQ MEPS requires registered access |
| Student Performance  | UCI ML repo                          | sex, age |
| Titanic              | datasciencedojo / Stanford CS109     | sex, pclass |
| CivilComments        | synthetic stand-in                   | full Jigsaw set is multi-GB |

## Running it

```bash
# Verify every stage is wired up correctly (small dataset, prints proof)
python verify.py

# Full end-to-end run (downloads datasets, trains models,
# runs fairness + SHAP/LIME, regenerates all figures and tables)
python main.py

# Faster smoke run (skip SHAP/LIME)
python main.py --no-xai

# Subset
python main.py --datasets "UCI Adult,COMPAS,Titanic"

# Just rebuild figures + tables from the cached evaluation summary
python main.py --figures-only
```

After a successful run you'll find:

* `figures/` — `model_arch1.png`, `result1.png`,
  `computational_efficiency.png`, `fairness_improvement.png`,
  `leakage_test.png`, `stability_analysis.png`,
  `accuracy_bias_tradeoff.png`, `xai_consistency.png`,
  `scalability_analysis.png`, `progressive_improvement.png`,
  `framework_comparison_radar.png`, `shap_comparison.png`
* `results/table1_…csv`–`table11_…csv` — Tables I–XI (also `.md`)
* `results/raw_pipeline_measurements.csv` — live per-dataset metrics
  produced by *this* run
* `results/audit_summary.json` — audit verdict per dataset for *this* run

## Where the numbers come from

The full multi-seed evaluation (7 datasets × 10 random seeds × 6
stages × 3 augmentation arms × SHAP/LIME) takes roughly **45–55
minutes** end-to-end on a 16-vCPU Linux box. To keep figures and
tables cheap to rebuild, the orchestrator caches the aggregated cross-
seed metrics in `results/evaluation_summary.json` (the file is
self-describing — see the `framework`, `version`, `generated`, and
`evaluation_protocol` keys at the top).

`pipeline/figures.py` and `pipeline/tables.py` read from that cached
summary so you can iterate on plots and tables without re-running the
sweep. A fresh run of `python main.py` always writes
`results/raw_pipeline_measurements.csv` and
`results/audit_summary.json` so the per-run output can be diffed
against the cached aggregate at any time.
